# 抖音三联图分类技术方案

## 一、需求理解

参考截图效果：
- 一张图片被**垂直切割**成 3 等份
- 每份呈现为一个独立的"抖音视频卡片"
- 每个卡片包含：黄色"置顶"角标、播放量数字（静态纯视觉，无播放功能）
- 三个卡片并排，整体构成一个作品的完整展示
- 瀑布流中：三联缩略展示；灯箱中：三联完整展示（带抖音UI元素，全部静态）

## 二、数据架构

### 2.1 data.json 新增分类

```json
{
  "id": "cat_triptych_xxx",
  "label": "抖音三联图"
}
```

### 2.2 三联图作品数据结构

```json
{
  "id": 1780xxx,
  "src": "img/抖音三联图/作品名_full.webp",
  "caption": "001 作品名",
  "category": "cat_triptych_xxx",
  "type": "triptych",
  "thumbnail": "img/抖音三联图/thumb_作品名.webp",
  "slices": {
    "left": "img/抖音三联图/作品名_left.webp",
    "mid": "img/抖音三联图/作品名_mid.webp",
    "right": "img/抖音三联图/作品名_right.webp"
  }
}
```

字段说明：
- `type: "triptych"` — 标识为三联图类型，区别于 image/video
- `slices` — 三个切片文件路径，用于渲染和灯箱展示
- `src` — 保留完整原图，灯箱中作为兜底/备用

### 2.3 文件夹结构

```
img/抖音三联图/
├── xxx_full.webp       # 完整原图（上传原图转WebP）
├── xxx_left.webp       # 左切片（1/3宽度）
├── xxx_mid.webp        # 中切片（1/3宽度）
├── xxx_right.webp      # 右切片（1/3宽度）
└── thumb_xxx.webp      # 缩略图（完整图等比缩放）
```

## 三、上传处理流程（upload_server.py）

### 3.1 自动切割逻辑

```python
def cut_triptych(img_path, out_dir, basename):
    """将图片垂直切割成3等份"""
    from PIL import Image
    img = Image.open(img_path)
    w, h = img.size
    
    # 垂直切割：每份宽度 = w // 3
    slice_w = w // 3
    slices = {
        'left':  (0, 0, slice_w, h),
        'mid':   (slice_w, 0, slice_w * 2, h),
        'right': (slice_w * 2, 0, w, h),
    }
    
    paths = {}
    for name, box in slices.items():
        cropped = img.crop(box)
        out_path = os.path.join(out_dir, f"{basename}_{name}.webp")
        cropped.save(out_path, 'WEBP', quality=WEBP_QUALITY)
        paths[name] = out_path
    
    return paths
```

### 3.2 上传流程（选择"抖音三联图"分类时）

1. 用户拖拽上传图片（建议尺寸：宽度为高度的 1.6875 倍，即 27:16，正好是 3 个 9:16）
2. 转 WebP → 保存 `xxx_full.webp`
3. 调用 `cut_triptych()` → 生成 `xxx_left.webp` / `xxx_mid.webp` / `xxx_right.webp`
4. 生成缩略图 `thumb_xxx.webp`
5. data.json 写入 `type: "triptych"` + `slices` 字段
6. 自动编号、caption 命名

### 3.3 尺寸建议提示

上传工具界面中，当选择"抖音三联图"分类时，提示用户：
> "建议上传 1215×720（27:16）或更宽的横图，以获得最佳三联效果"

## 四、瀑布流渲染（render.js）

### 4.1 三联图卡片 DOM 结构

```html
<div class="waterfall-item triptych-item" data-type="triptych" ...>
  <div class="triptych-preview">
    <img src="left.webp" class="triptych-slice" alt="">
    <img src="mid.webp" class="triptych-slice" alt="">
    <img src="right.webp" class="triptych-slice" alt="">
  </div>
  <!-- 悬停遮罩 -->
  <div class="item-overlay">
    <span class="item-caption">001 作品名</span>
    <button class="item-zoom">&#11170;</button>
  </div>
</div>
```

### 4.2 关键样式

- `.triptych-preview`：flex 布局，3个子元素平分宽度，高度一致
- `.triptych-slice`：每张图占 33.33%，object-fit: cover
- 切片之间有微小间隙（2px）+ 分割线，模拟三个独立卡片
- 卡片整体圆角，但内部切片需要特殊处理圆角

### 5.2 抖音UI元素设计

每个 `.triptych-card` 包含（**全部静态，无点击/播放交互**）：
1. **左上角黄底黑字"置顶"角标**：`background: #FACE15; color: #000; border-radius: 0 0 4px 0;`
2. **底部播放量信息栏**：半透明黑底 + 白色播放图标 + 播放量数字（纯视觉装饰，无功能）
3. **卡片圆角**：`border-radius: 8px; overflow: hidden;`
4. **卡片阴影**：模拟抖音视频卡片的层次感

### 5.3 灯箱布局

- 三个卡片水平排列，居中对齐
- 每个卡片宽度：建议 280px（桌面端），移动端缩放为 30vw
- 卡片间距：12px
- 整体最大宽度：限制在 90vw 内
- 支持左右箭头切换上一张/下一张作品

## 八、关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 切割方向 | 垂直三等分（左/中/右） | 截图明确为竖向三联 |
| 播放量数字 | 算法生成（基于作品ID的伪随机，范围30-999） | 无需用户输入，每次一致 |
| 瀑布流中展示 | 3个切片并排在一个卡片内 | 保持"一个作品"的语义，不拆成3个作品 |
| 灯箱中展示 | 3个带抖音UI的卡片并排 | 还原截图参考效果 |
| 是否保留完整原图 | 是，保存 `_full.webp` | 便于后续重新处理/备份 |
| 缩略图策略 | 使用完整图缩放为缩略图 | 瀑布流中展示三联整体效果 |
| 卡片宽高比 | 瀑布流中保持原图比例；灯箱中强制 9:16 | 瀑布流自适应，灯箱还原抖音风格 |

## 九、播放量算法（伪随机但固定）

```javascript
function getPlayCount(seed, index) {
  // seed = 作品ID，index = 0/1/2（左中右）
  var n = ((seed * 9301 + 49297) % 233280) / 233280;
  var base = [30, 100, 20][index]; // 左/中/右的基础值
  return Math.floor(base + n * 900);
}
```

这样左卡片播放量 30~300，中卡片 100~1000，右卡片 20~500，符合截图中的分布规律（37, 343, 44）。
