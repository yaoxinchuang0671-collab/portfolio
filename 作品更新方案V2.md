# 潮汐视界作品集 — 作品更新方案 V2（一键入库版）

> 基于现有架构的全新作品更新流程，替代原有的 Web 上传压缩方案。
> **核心改进**：本地上传工具一键完成"文件处理 + 自动编号 + 写入 data.json + 分类归档"，无需再手动去后台填路径。

---

## 一、现状分析

### 原方案问题
| 问题 | 影响 |
|------|------|
| 前端压缩 + 服务端转换，链路长 | 大 PSD/视频处理慢，容易超时 |
| 上传代码分散在 admin.html + server.py | 维护困难，一处改动处处崩 |
| 缩略图、重命名、文件类型判断交织 | 逻辑复杂，bug 难定位 |
| 新增作品需要"上传文件 → 后台填路径 → 保存"三步 | 步骤繁琐，效率低 |

### 现有架构（保留）
- `data.json` — 作品元数据（标题、分类、路径、缩略图）
- `img/` — 作品图片存储目录
- `build.py` — 构建脚本（压缩代码到 `build/`）
- `upload_github_api.py` — 上传代码到 GitHub Pages
- `upload_cos.py` — 上传图片到腾讯云 COS
- `admin.html` — 后台管理（编辑元数据、排序、删除、发布）

---

## 二、新方案核心思想

**浏览器拖拽上传**：打开上传工具页面，拖拽文件、选择分类、一键入库。

```
打开浏览器 → 拖拽文件 → 选择分类 → 一键上传
    ↓
自动处理 → WebP + 缩略图 + 自动编号 + 按分类归档 + 写入 data.json
    ↓
刷新浏览器后台，作品已在列表中（只需微调标题/排序）
    ↓
python build.py + python upload_github_api.py
    ↓
部署完成
```

## 三、文件格式处理策略（不限制时长）

### 3.1 静态图片（JPG/PNG/WebP/BMP/TIFF）

**处理**：PIL 读取 → 转 WebP（quality=85）→ 生成 400x400 缩略图

**输出**：
- `img/分类名/XXXX_名称_时间戳.webp`
- `img/分类名/thumb_XXXX_名称_时间戳.webp`

---

### 3.2 GIF 动图

**处理**：PIL 读取所有帧 → 动态 WebP（**保留完整帧数和时长，不限制**）→ 缩略图取第一帧

**输出**：动态 WebP + 静态缩略图

---

### 3.3 PSD 文件

**处理**：psd-tools 合并所有图层 → WebP（quality=85）→ 生成缩略图

**注意**：大 PSD 处理可能较慢，但本地运行无超时限制。

---

### 3.4 视频（MP4/MOV/AVI/MKV/WebM/M4V/FLV）

**处理**：ffmpeg 完整转动态 WebP（**不限制时长，保留完整内容**）
- 15fps，宽度 720px 保持比例
- 质量 85

**输出**：动态 WebP + 视频第一帧作为缩略图

**注意**：长视频转动态 WebP 文件可能较大，建议根据实际使用场景选择是否用视频原文件直传（修改脚本参数即可）。

## 四、使用方法

### 前置准备

```bash
# 确保依赖已安装（只需一次）
python -m pip install Pillow psd_tools imageio imageio-ffmpeg
```

### 方式一：浏览器拖拽上传（推荐）

**步骤 1：启动上传工具**

```bash
cd img
python upload_server.py
```

或双击 `img/start_upload.bat`

会自动打开浏览器访问 `http://localhost:8765`

**步骤 2：拖拽上传**

1. 在页面上方**选择分类**（下拉框自动读取 data.json 分类）
2. 将文件**拖拽到上传区域**（支持多文件批量）
3. 点击 **「开始上传」**
4. 等待处理完成，页面显示成功/失败结果

**上传工具功能**：
- 支持拖拽 + 点击选择文件
- 支持多文件批量上传
- 自动转 WebP、生成缩略图
- 自动编号（从当前最大编号 +1 开始）
- 自动按分类保存到 `img/分类名/` 文件夹
- 自动追加到 `data.json`

**步骤 3：后台微调**

1. 打开 `http://localhost:8080/admin.html`
2. 新作品已自动出现在列表末尾
3. 可拖拽排序、修改标题、调整分类
4. 点击「发布代码」或「全部发布」

---

### 方式二：命令行自动入库（批量/无人值守）

将待上传的作品文件放入 `img/inbox/` 目录：

```
img/inbox/
├── 作品1.jpg          # 无子文件夹 = 使用默认分类
├── 作品2.png
├── 小程序/
│   ├── 作品3.gif      # 放在"小程序"文件夹 = 自动匹配"小程序"分类
│   └── 作品4.psd
└── 海报设计/
    ├── 作品5.mp4
    └── 作品6.mov
```

然后运行：

```bash
# 预览模式（不实际处理，先看效果）
python auto_upload.py --dry-run

# 正式入库
python auto_upload.py

# 指定默认分类（用于根目录文件）
python auto_upload.py --default-cat="小程序"
```

**分类匹配规则**：
- 子文件夹名与 `data.json` 中的分类 `label` **精确匹配**或**包含匹配**
- 匹配失败或无子文件夹 → 使用默认分类

---

### 步骤 3：构建部署

```bash
python build.py              # 构建到 build/
python upload_github_api.py  # 上传代码到 GitHub Pages
python upload_cos.py         # 上传图片到腾讯云 COS
```

## 六、工具脚本

### 上传工具（浏览器拖拽）

| 文件 | 路径 | 说明 |
|------|------|------|
| `upload_server.py` | `img/upload_server.py` | 本地服务器，处理文件上传 |
| `upload_tool.html` | `img/upload_tool.html` | 上传界面，拖拽上传 |
| `start_upload.bat` | `img/start_upload.bat` | 双击启动脚本 |

**启动方式**：
```bash
cd img
python upload_server.py    # 自动打开浏览器
```

**工作原理**：
1. 服务器读取 `data.json` 获取分类列表
2. 前端页面显示分类下拉框
3. 用户拖拽文件 → 前端发送到 `/upload`
4. 服务器处理：转 WebP → 生成缩略图 → 保存到 `img/分类名/`
5. 自动编号，追加写入 `data.json`
6. 返回处理结果

### auto_upload.py（命令行批量）

```bash
python auto_upload.py              # 正式入库
python auto_upload.py --dry-run    # 预览模式
python auto_upload.py --default-cat="小程序"  # 指定默认分类
```

**特点**：
- 纯命令行，无 Web 依赖
- 错误隔离：单文件失败不影响其他文件
- 自动分类：子文件夹名智能匹配
- 源文件归档：处理完移到 `.processed/`，防止重复处理

## 八、与旧方案对比

| 维度 | 旧方案（Web 上传） | 新方案（上传工具） |
|------|-------------------|-------------------|
| 新增作品步骤 | 拖拽 → 等待压缩 → 填标题 → 保存 | 拖拽 → 选分类 → 上传 → 完成 |
| 操作位置 | 浏览器后台 | 浏览器上传工具页面 |
| 大 PSD 支持 | 容易超时（300s） | 本地处理，无超时 |
| 视频/GIF 时长 | 限制 5 秒 | **不限制，完整保留** |
| 批量处理 | 前端循环 fetch，易失败 | 一次上传多个文件 |
| 分类归档 | 所有文件在 img/ 根目录 | **自动按分类放入子文件夹** |
| 自动编号 | 需后台手动填写 | **自动编号** |
| 出错排查 | 浏览器+服务器两端日志 | 单端命令行输出 |
| 维护成本 | 高（前端+后端 1200+ 行代码） | 低（独立工具 ~300 行） |

## 九、文件命名规范

脚本自动命名格式：`{编号}_{安全名称}_{时间戳}.webp`

示例：
- `0340_暴雪世界杯_1780901244558.webp`
- `thumb_0340_暴雪世界杯_1780901244558.webp`

**编号规则**：
- 从当前 data.json 中最大编号 +1 开始
- 4 位数字，不足补零（0001, 0010, 0340）
- 标题默认使用 `"编号 安全名称"`（如 `"0340 暴雪世界杯"`）

## 十、注意事项

1. **inbox 目录不要提交到 Git** — 建议加入 `.gitignore`
2. **.processed 归档** — 处理完的源文件会移到 `img/inbox/.processed/`，可定期清理
3. **动态 WebP 兼容性** — 部分旧版 Safari 不支持，但 Chrome/Firefox/Edge 已完全支持
4. **长视频文件大小** — 完整视频转动态 WebP 可能产生大文件（数十 MB），若需控制大小可修改脚本中的 `scale=720` 为更小值
5. **备份** — 处理前原文件在 inbox，处理后在 .processed，双重保障
6. **上传工具端口** — 默认 8765，如被占用可修改 `upload_server.py` 中的 `PORT`
