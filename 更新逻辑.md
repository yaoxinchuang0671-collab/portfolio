# 潮汐视界 更新逻辑梳理

## 一、项目结构

```
portfolio/                    # 源代码目录
├── index.html                # 首页
├── admin.html                # 后台管理
├── ledger.html               # 账本（单文件 Vue 应用，不压缩）
├── styles.css / gallery.css  # 样式
├── render.js / gallery.js    # 首页逻辑
├── main.js / sw.js           # 入口 & Service Worker
├── data.json                 # 首页作品数据
├── ledger_data.json          # 账本数据（本地文件）
├── img/                      # 图片资源（原图 + 缩略图）
├── js/                       # 额外脚本
├── build.py                  # 构建脚本
├── upload_github_api.py      # GitHub Pages 上传
├── upload_cos.py             # 腾讯云 COS 上传
├── server.py                 # 本地开发服务器
└── build/                    # 构建产物（由 build.py 生成）
```

---

## 二、本地开发流程

### 1. 启动本地服务器
```bash
cd portfolio
python server.py
# 监听 localhost:8080
# 支持 POST /save-ledger（账本保存到本地文件）
```

### 2. 修改文件后直接刷新浏览器
- 首页：`http://localhost:8080/index.html`
- 账本：`http://localhost:8080/ledger.html`
- 后台：`http://localhost:8080/admin.html`

### 3. 账本数据存储
- **本地**：`fetch('/ledger_data.json')` 直接读取本地 JSON 文件
- **保存**：点击"保存到本地" → `POST /save-ledger` → 写入 `ledger_data.json`
- **强制刷新**：从 GitHub 拉取 → 自动 `POST /save-ledger` 同步到本地文件

## 三、构建流程（build.py）

```bash
python build.py
```

### 构建规则
| 文件类型 | 处理方式 | 说明 |
|---------|---------|------|
| `.css` | 压缩（去注释、去空白） | 缩减体积 |
| `.js` | 压缩（去注释、去空白） | 缩减体积 |
| `.html` | 压缩（去注释、去空白） | 缩减体积 |
| `ledger.html` | **直接复制** | 单文件 Vue，压缩会破坏模板语法 |
| `data.json` | 直接复制 | 数据文件 |
| 其他 | 直接复制 | |

### 输出
- 清空并重建 `build/` 目录
- 代码文件压缩后体积通常减少 **30%~50%**

### ⚠️ 注意
`build/` **不包含** `img/` 和 `js/` 目录，上传前需要手动复制：
```bash
cp -r img build/img
cp -r js build/js
```

## 四、上传流程

### 双平台部署架构

```
┌─────────────────────────────────────────┐
│              用户访问入口                 │
│   GitHub Pages: yaoxinchuang0671-collab  │
│          .github.io/portfolio/           │
│         （HTML/JS/CSS/JSON 代码）         │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│          图片资源加载（CDN）              │
│   腾讯云 COS: portfolio-images-xxx      │
│       .cos-website.ap-guangzhou         │
│        .myqcloud.com/portfolio/         │
│              （图片文件）                 │
└─────────────────────────────────────────┘
```

### 4.1 GitHub Pages 上传（代码）

```bash
# 默认模式：只传代码文件（.html/.js/.css/.json/.md）
python upload_github_api.py

# 全部更新：代码全传 + 有变化的图片（对比 SHA-256 manifest）
python upload_github_api.py --force

# 仅图片：只上传有变化的静态资源
python upload_github_api.py --images
```

**上传逻辑**：
1. 优先读取 `build/` 目录，没有则读源码目录
2. 代码文件**总是上传**（无论是否变化）
3. 图片等静态文件**比对 manifest**：SHA-256 相同则跳过
4. 上传成功后**自动清理** `build/` 目录
5. 保存 `.upload_manifest.json` 记录已上传文件哈希

### 4.2 腾讯云 COS 上传（图片）

```bash
# 增量上传图片（只传新图/变化图）
python upload_cos.py

# 强制重传所有图片
python upload_cos.py --force

# 上传静态网站文件到 COS
python upload_cos.py --website

# 清空 COS 图片
python upload_cos.py --clean

# 清空 COS 网站文件
python upload_cos.py --clean-website
```

**上传逻辑**：
1. 扫描 `img/` 目录下所有图片（.jpg/.jpeg/.png/.webp/.gif/.avif）
2. 列出 COS 上 `portfolio/` 前缀的已有文件
3. **文件名存在且非 force 模式** → 跳过
4. **新文件或 force 模式** → 上传

## 五、缓存策略

### 浏览器缓存（首页代码）
在 `index.html` 中通过 **query string 版本号**强制刷新：
```html
<link rel="stylesheet" href="styles.css?v=30" />
<script src="render.js?v=35"></script>
<script src="gallery.js?v=25"></script>
```

**何时提升版本号**：
- JS/CSS/HTML 文件内容修改后
- 在 `index.html` 中修改对应引用版本号
- 重新构建并上传

### Service Worker 缓存（sw.js）
```javascript
const CACHE_NAME = 'portfolio-cache-v7';
```
修改 sw.js 逻辑后提升版本号，触发客户端重新注册。

### COS 图片缓存
- 图片上传到 COS 后 URL 固定
- 同名文件覆盖后 CDN 可能有延迟（通常几分钟）
- 新图片直接上传即可

## 六、数据更新流程

### 6.1 首页数据（data.json）
```
admin.html 修改 → data.json 更新 → build.py → upload_github_api.py
```
- 后台添加/编辑/删除作品 → 自动保存到 `data.json`
- 运行 `build.py` 生成压缩版
- 运行 `upload_github_api.py` 上传代码

### 6.2 账本数据（ledger_data.json）

**本地开发**：
```
ledger.html 操作 → POST /save-ledger → 写入 ledger_data.json
```

**线上环境**：
```
ledger.html 操作 → localStorage 存储
                    ↓
            点击"上传到 GitHub" → GitHub API 更新
                    ↓
            点击"强制刷新" → GitHub Raw 拉取 → localStorage
```

## 七、完整更新 checklist

### 场景 1：修改首页代码（JS/CSS/HTML）
```bash
# 1. 本地调试确认无误
# 2. 提升 index.html 中的版本号
# 3. 构建
python build.py
# 4. 复制资源目录
cp -r img build/img
cp -r js build/js
# 5. 上传代码
python upload_github_api.py
```

### 场景 2：添加/修改作品图片
```bash
# 1. 确保图片在 img/ 目录（原图 + thumb_ 缩略图）
# 2. data.json 已更新
# 3. 构建 + 复制
cp -r img build/img
# 4. 上传代码（data.json 有变化）
python upload_github_api.py
# 5. 上传新图片到 COS
python upload_cos.py
```

### 场景 3：账本数据更新
```bash
# 本地：
#   直接操作 ledger.html，自动保存到 ledger_data.json
# 线上：
#   点击"上传到 GitHub" 或"强制刷新"同步
```

### 场景 4：全量更新（代码 + 图片）
```bash
# 1. 构建
python build.py
cp -r img build/img
cp -r js build/js
# 2. 上传代码 + 图片到 GitHub
python upload_github_api.py --force
# 3. 上传图片到 COS（如有新图）
python upload_cos.py
```

## 八、常见问题

| 问题 | 原因 | 解决 |
|-----|------|------|
| 线上页面没变化 | 浏览器缓存旧版本 | 提升版本号 `?v=xx` 后重新上传 |
| 图片 404 | COS 上缺少该图片 | 运行 `python upload_cos.py` |
| 账本保存失败 | 本地服务器未启动或不是 server.py | 用 `python server.py` 启动 |
| 作品空白/黑底 | gallery.js masonry 布局问题 | 检查 `waitImagesAndReflow()` |
| 加载屏不消失 | render.js 异常未捕获 | 加 `try...finally` 兜底 |
